MangoDjango

The driver incentive app was a CPSC 4910 (Senior Computing Practicum) project that was designed to help sponsors encourage drivers to be safer on the road. Good driving is rewarded with a point system that allows drivers to exchange their points for items their sponsor would provide.

Our project was developed and deployed using the Django framework and a MySQL database. Our UI was built with Bootstrap and we secured user authentication using AWS Cognito. The entire infrastructure was hosted on an AWS RDS instance and we made weekly commits through Azure DevOps. The project was created within a 4 month period and practiced AGILE Scrum style weekly meeting and planning. Every week consisted of delegated user stories and tasks through Azure DevOps.

My work in this project heavily consisted of creating a catalog of products that made use of Spotify's API, developing account creation using Cognito, creating an order system, creating an ticket report system and various webpages for extracting sales and order information.

Challenges

The project had 2 major challenges that we faced. The first was establishing each type of user and their complexity. There were 3 type of users and thus created a few layers of depth each user would require: drivers, sponsors, and admins. Every sponsor had a list of driver users asigned to monitor (change points, see tickets, add or suspend driver), reject and approve purchase orders, and view total sales in purchases. Meanwhile, driver users could have multiple sponsors (in case a driver had more than 1 job or company they worked for), could redeem their points for a reward from their sponsor's catalog, and request to leave a sponsor.

The second challenge was improving security on our website. We used AWS Cognito to secure user authentication and password resetting. Resetting would involve receiving a code through the account's provided email to reset the account's password. We also used CSRF/SQLi protection to strip user input from injecting malicious code or commands.

Professional Development Skill

One of our requirements was to go above and beyond meeting the specifications for the app. This meant the creative freedom to develop a useful tool of our choice. An example could be making our web app a mobile app. For this requirement, I developed a ticketing system that all users could use to communicate with each other. Every user had a ticket dashboard. If a driver noticed their points missing or wants to rectify an adjustment to their orders, they could create a ticket and select what kind of issue they are having. Their ticket would be recevied by their selected sponsor and the sponsor would be able to reply back and update the status of the tickets. If sponsors have issues outside their technical capabilities, they could also create tickets that admins would receive. In short, I created a ticketing system with a message chat where users could communicate back and forth.

Other project notes:
  • The project did not involve using any kind of GPS tracking or conditions to obtain points. In a real scenario, the sponsor would set the rules and conditions for rewarding points for good driving behavior.
  • Spotify's API was free to use and used as the catalog to search and "purchase" an item.
